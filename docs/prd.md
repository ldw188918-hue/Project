# 제품 요구사항 문서 (PRD)
## Digital Twin 기반 공급망 리스크 분석 대시보드

### 1. 프로젝트 개요
*   **프로젝트명**: Digital Twin 기반 공급망 리스크 분석 대시보드
*   **타겟 사용자**: 한화시스템 / 제조 분야 의사결정권자
*   **배경**:
    *   한화시스템의 스마트 팩토리(MES) 및 ERP 연동 사업 강점 부각.
    *   글로벌 공급망 불안정을 ICT 솔루션으로 해결하고자 하는 니즈.
    *   단순 데이터 시각화를 넘어선 **의사결정 지원 시스템(DSS)**으로서의 가치 증명.

### 2. 목표
*   **핵심 목표**: 부품 수급 차질(단가 상승, 납기 지연 등)이 생산 라인에 미치는 영향을 시뮬레이션하고 시각화하는 웹 기반 대시보드 개발.
*   **가치 제안**:
    *   **DSS 역량**: 전략적 의사결정의 핵심 지표(KPI) 정의 및 모니터링.
    *   **비용 효율성**: Python 오픈소스 생태계(Pandas, Plotly)를 활용한 고효율 대시보드 구축.
    *   **전략적 통찰**: 다양한 시나리오의 결과를 예측하는 'What-If 분석' 기능 제공.

### 3. 주요 기능
#### 3.1. 대시보드 시각화 (디지털 트윈 뷰)
*   **공급망 맵**: 공급업체, 물류 경로, 재고 수준의 시각적 표현.
*   **생산 현황**: 생산 라인 가동률 및 병목 구간 실시간(시뮬레이션) 모니터링.
*   **리스크 지표**: 재고 부족, 선적 지연, 고위험 공급사 알림(히트맵 등).

#### 3.2. What-If 분석 (시뮬레이션)
*   **시나리오 빌더**: 사용자가 다음 변수를 조절 가능:
    *   원자재 단가 변동률 (+/- %).
    *   공급업체 납품 지연 일수 (Days).
    *   생산 라인 설비 가동 중단.
*   **영향도 평가**: 다음 항목의 변화를 자동 계산 및 표시:
    *   **영업이익**: (예: "부품 A 단가 20% 상승 시 영업이익 $X 감소").
    *   **생산량**: 목표 대비 생산 차질 물량.
    *   **납기 일정**: 최종 완제품 인도 지연 가능성.

#### 3.3. KPI 및 DSS 지원
*   **KPI 정의**:
    *   *서비스 수준(Service Level)* vs *재고 비용*.
    *   *리스크 노출 지수(Risk Exposure Index)*.
    *   *생산 처리 효율성(Throughput Efficiency)*.
*   **인사이트 도출**: 시뮬레이션 결과에 기반한 추천 (예: "마진 유지를 위해 공급사 B로 전환 권장").

### 4. 기술 아키텍처
*   **언어**: Python 3.x
*   **데이터 처리**: Pandas (전처리), NumPy (시뮬레이션 로직).
*   **시각화**: Plotly (인터랙티브 차트), Plotly Express.
*   **웹 프레임워크**: Streamlit (대시보드 호스팅 및 UI 인터랙션).
*   **데이터 소스**: Kaggle 제조/공급망 데이터 활용 (또는 합성 데이터 생성).

### 5. 구현 전략
1.  **데이터 준비**: 정교한 합성 데이터 생성 또는 Kaggle 데이터셋 정제.
2.  **로직 개발**: 'What-If' 시나리오 계산 엔진 구현.
3.  **UI/UX 디자인**: 가독성과 의사결정 지원에 초점을 맞춘 전문적인 레이아웃.
4.  **통합**: 로직과 UI를 결합하여 로컬 웹 앱으로 배포.

### 6. 성공 지표
*   **반응 속도**: 입력 값 변경 후 3초 이내 시뮬레이션 결과 업데이트.
*   **사용성**: "Before vs After" 시나리오의 명확한 시각화.
*   **연관성**: 영업이익/손실 등 재무적 영향도(Financial Impact) 입증.
